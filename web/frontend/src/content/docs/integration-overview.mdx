## Integration Modes

### Mode A: Server-Validated

Use when you have a backend server that controls access to resources.

- Client requests protected resource
- Server returns 402 with payment requirements
- Client signs and retries
- Server verifies payment via Facilitator

**Best for:** APIs, premium content, SaaS features

---

### Mode B: Serverless

Use when you want direct browser-to-facilitator payments without a backend.

- Client calls Facilitator directly
- No server needed
- Ideal for static sites, SPAs

**Best for:** Donations, tips, simple payments, dApps
