## @x402x/extensions

The core extension library that adds programmable settlement to standard x402 V2.

- **Hook Helpers**: `TransferHook.getAddress(network)`, `TransferHook.encode(splits?)`
- **Configuration**: `createSettlementRouteConfig` (for Server)
- **Schemes**: `ExactEvmSchemeWithRouterSettlement` (for Client)
- **Registration**: `registerRouterSettlement(server)` (for Hono/Express)
- **Network Helpers**: `getSupportedNetworkIds()`

## @x402x/client

Serverless client SDK for interacting directly with the Facilitator (no Resource Server required).

- Class: `new x402xClient({ wallet, network, facilitatorUrl, ... })`
- Methods: `execute({...})`
- Primitives: `prepareSettlement`, `signAuthorization`, `settle`

## Official x402 SDKs

We use the official packages from `@x402/*`.

### @x402/hono
Official middleware for Hono.
- `x402ResourceServer`: The main server class.
- `paymentMiddleware`: The route guard.

### @x402/client
Official Browser/Node client.
- Can be used with `ExactEvmSchemeWithRouterSettlement` from `@x402x/extensions`.

## Legacy Packages (Deprecated)

The following packages are v1 wrappers/forks and are **deprecated** in favor of official SDKs + extensions:

- `@x402x/core`: Replaced by `@x402x/extensions`.
- `@x402x/fetch`: Use `@x402/client`.
- `@x402x/hono` / `@x402x/express`: Use `@x402/hono` / `@x402/express` + extension registration.
